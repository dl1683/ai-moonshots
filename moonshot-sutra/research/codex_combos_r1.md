Bottom line: the document overstates coherence for Combo 2, 3, and 4. Among the five as written, Combo 2 has the best real signal-sharing coherence, Combo 5 has the best chance to ship and actually win something, and Combo 3 should be downgraded materially after the OT retrieval loss.

The key file-grounded reasons are in [STAGE_ANALYSIS](/C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-sutra/research/STAGE_ANALYSIS.md#L1000), [STAGE_ANALYSIS](/C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-sutra/research/STAGE_ANALYSIS.md#L944), [STAGE_ANALYSIS](/C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-sutra/research/STAGE_ANALYSIS.md#L969), and [STAGE_ANALYSIS](/C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-sutra/research/STAGE_ANALYSIS.md#L1038).

1. Combo 1, Conservative: coherence is `medium-high`, not high. It is compatible, but not truly synergistic. Hidden issue: `A1b` token patches make the meaning of `A3a` “GRU within patches” less clean; the GRU is no longer obviously processing the right atomic unit. Chance of beating a matched transformer: `2/10`. Kill test: one matched-compute run against a tokenized transformer and current v0.4; if BPB and retrieval are still clearly behind, retire it.

2. Combo 2, Kalman Machine: this has the best genuine coherence, but the doc slightly oversells it. The real shared signal is only `Stage 5 -> 6 -> 7`; the claim that `A2c` uses uncertainty is not supported by the Stage 2 formulation, which is content/global-summary based, not variance-based in the text [STAGE_ANALYSIS](/C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-sutra/research/STAGE_ANALYSIS.md#L969). Hidden issue: the proposed Kalman update only decreases variance, so variance may track “how many updates happened” more than “am I correct?” [STAGE_ANALYSIS](/C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-sutra/research/STAGE_ANALYSIS.md#L940). Chance of beating a matched transformer: `3/10`. Kill test: variance-vs-correctness AUROC/ECE on held-out next-token errors; if variance is not better than simple logit margin as a correctness predictor, kill the Kalman path.

3. Combo 3, Transport Network: coherence is `medium` at best, not medium-high. Hidden conflicts: dynamic segmentation destabilizes hierarchical addresses; OT budgets can hurt cases where many targets legitimately need the same source; and `A7d` verify-loop clashes with monotone variance reduction, because verify failure should increase uncertainty, not only decrease it [STAGE_ANALYSIS](/C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-sutra/research/STAGE_ANALYSIS.md#L432), [STAGE_ANALYSIS](/C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-sutra/research/STAGE_ANALYSIS.md#L944). Chance of beating a matched transformer: `1/10`. Kill test: a scarce-source exact-retrieval benchmark with raw `A3b` features and matched FLOPs vs attention; if OT loses again, stop pursuing pure-OT Stage 4.

4. Combo 4, Biological: coherence is `medium`, not high. The claimed “variance tells routing where to invest effort” link is not actually specified in `A4d`; grown sparsity is described as evolving from gradient usefulness, not from variance. Hidden issue: `A1d` learned boundaries and `A4d` grown routing are both slow latent structures, so you get a bad co-adaptation loop early in training. Chance of beating a matched transformer: `2/10`. Kill test: track routing-table emergence and retrieval gain over training; if structure forms slowly or not at all while performance lags dynamic sparse routing, kill it.

5. Combo 5, Pragmatist: coherence is `medium-high`. It is the best engineering package, but not because stages “lift” each other strongly; mostly because nothing is too exotic. Hidden issue: the verifier is downstream and mostly rescues ranking, not representation. If the right answer is not in the top candidates, `A7c` does nothing. PonderNet is also already known to be fragile in this repo. Chance of beating a matched transformer: `4/10`. Kill test: one end-to-end run with verifier reranking included in total compute; if it does not beat Combo 1 clearly on BPB plus hard-task accuracy, it is not worth the added complexity.

Best coherence among the five: `Combo 2`, but only because `5 -> 6 -> 7` genuinely share one state variable. Best chance to actually beat a standard transformer: `Combo 5`, because it makes the fewest risky bets. I would still call that an under-50% shot at matched scale.

The missing sixth combo is a hybrid Stage-4 design, not a purist moonshot:
`A1b + A2d + A3b + A4e + A5c + A6b + A7c`
Stable token patches, hierarchical addressing, stronger local features, hybrid routing instead of pure OT, Kalman only where it creates a real shared control signal, and verifier reranking instead of a full decode-verify loop. This removes the three worst conflicts in Combo 3: unstable segmentation, pure-OT single-point failure, and the variance/verify-loop inconsistency. Chance of beating a matched transformer: `5/10`. This also matches the repo’s separate Stage-4 conclusion that the realistic path is hybrid global refresh rather than pure routing ideology [codex_pipeline_r4](/C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-sutra/research/codex_pipeline_r4.md#L31), [codex_pipeline_r4](/C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-sutra/research/codex_pipeline_r4.md#L76).

On sequence: `Pragmatist -> Kalman -> Novel` is right for shipping, but wrong for deciding whether Sutra can beat transformers. If the real question is “can this architecture family win?”, I would do:
1. `Pragmatist`
2. `Stage-4 killer experiment` for OT/hybrid routing
3. `Sixth combo`
4. `Kalman integration if variance is real`
5. `Pure Combo 3/4 only if Stage 4 wins first`

The TinySutra OT result absolutely changes Combo 3. `39.5% vs 46.5%` on retrieval is not a side note; retrieval is the exact subproblem OT is supposed to justify. The documented mean-pooling failure means the result is not final, but it does mean Combo 3 should move from “high-risk moonshot” to “not on critical path until it re-wins the toy domain.” If pure OT cannot beat attention on controlled retrieval, it is very unlikely to beat a matched transformer in language modeling.
